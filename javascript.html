Variables

Variables can be thought of as containers for storing data. Variables are named to allow us to access and refer to the data later. For example let x = 10 is a statement that creates a container named x that stores a piece of data, in this case, the number 10.

Below we are creating three variables using three different keywords var, let and const.

  • Variables created with var can be changed and accessed pretty much everywhere.
  • Variables created with let can be changed and accessed only within the block (more on this later) where they are created.
  • And, lastly, variables created with const cannot be changed and like let can only be accessed within the block where they are created.

In the statements below, the variables are assigned values using the = (assignment) operator. If variables are not explicitly initialized with values when they are created, they have a default value of undefined. 


              
var age = 37

              
let userName = 'dorian'

              
const married = true

javascript variable names have to begin with a letter, an underscore '_', or a dollar sign '$'

Data Types

In javascript there are two different data types: primitive types, and object types. Primitives consist of:

  • numbers
  • strings
  • boolean
  • undefined
  • null

Object type data includes things like functions and arrays. An object is a collection of properties, each consisting of a name and a value.

Operators

Mathematical operators are also built into javascript. The most basic are addition +, subtraction -, multiplication *, division / and the modulus operator % which returns the remainder of a division operation. 


              
const x = 3 + 8 // x = 11

              
const y = 7 - 9 // y = -2

              
const z = 5 * 9 // z = 45

              
const a = 32 / 8 // a = 4 

              
const b = 26 % 3 // b = 2

ES2016 added the ** operator for exponentiation. (2**3 = 8)

Comparison Operators

Javascript also has built-in operators to compare values


              
== // equals

              
=== // strictly equals

              
< // less than

              
> // greater than

              
<= // less than or equal to

              
>= // greater than or equal to

              
!= // not equal to

The strictly equals operator checks to see if two values are exactly the same. It checks for both value and type equality. Javascript is very flexible about data types, it will convert data as needed. For example 3 + 'developers' will returns "3 developers". Likewise 34 == '34' returns true because javascript converts the the string 34 to the number 34. Strict equality does not do this; it doesn't attempt to convert data. So 34 === '34' returns false.

The unary operator ! indicates NOT. Think of it as a reverse operator. For example !true would evaluate as false and converse !false would evaluate as true.

Functions

Functions are javascript objects that can be thought of as chunks of reusable code. A function is defined (created) once and then called (invoked) any number of times. Functions in javascript can have parameters, optional pieces of data that are given to the function when it's called. This data is then used within the body of the function.

defining functions

Functions can be defined via function declarations or function expressions. The following is a function declaration. It takes the general form function name(parameter){body}.

We declare our function with the keyword function. It's followed by a mandatory identifier or name, in this case getAge; think of this as a variable that our function object is stored in. This function takes a parameter yearOfBirth that we use in the body of the function. Next is a pair of curly braces {} that encloses the body of our function. The body contains 0 or more statements that are executed when we invoke our function.

Next, our function has a return statement, an optional statement that returns a value to it's caller. Without an explicit return statement, invoking the function returns undefined. 


                

                  // function declaration 

                  function getAge(yearOfBirth){ 

                    let year = new Date().getFullYear() 

                    let result = year - yearOfBirth 

                    return result 

                  } 

                  getAge(1984) // 37

Note: multiple parameters would be separated by commas.

The following is a function expression. The first function is an anonymous function as it has no name. The second function is named and this name allows the function to refer to itself inside of itself. 


                

                  // anonymous function expression 

                  const f = function(yearOfBirth){ 

                     let year = new Date().getFullYear() 

                     let result = year - yearOfBirth 

                     return result 

                  } 

                  // named function expression 

                  const f = function getAge(yearOfBirth){ 

                     console.log(typeof getAge) // prints 'function' 

                     let year = new Date().getFullYear() 

                     let result = year - yearOfBirth 

                     return result

                  } 

                  console.log(typeof getAge) // prints 'undefined' as the getAge name only exists within the named function body

Function expressions are called using the variable they were stored in. In case of the two functions above, we would write f(1984), which, as of 2021, would return 37.

return statement

The return statement in a function is optional, and if it is included, returning a value is optional as well. A function can return a value, object, or even a function. It can also simply return, which means the function will end. Any code written below a return statement will be ignored as the function has ended.

Arrow Functions

Javascript arrow functions are a shorter way to write function expressions. 


              

                const add = function( a, b){ 

                   return a + b 

                }
              

              

                // can be re-written as: 

                const add = ( a, b) => a + b

A single line arrow function doesn't require an explicit return statement. Multi-line functions, however, like traditional functions, do. 


              const getAge = yearOfBirth => { 

                 let year = new Date().getFullYear() 

                 let result = year - yearOfBirth 

                 return result 

              } 

              getAge(1984) //37

Note: an arrow function that takes a single parameter doesn't require parenthesis.

Arrays

Arrays are an ordered collection of data. Each piece of data in the array is called an element and its position in the array is its index. Javascript arrays, like strings are zero-based meaning we start counting with 0. In the string 'cad bane' the position of the character 'c' is 0.

Javascript arrays are also dynamic meaning they grow and shrink if data is added or deleted. They are also untyped which means they don't have a specific data type and each element can be of a different data type. E.g let arr = [ 1, true, 'string', {name: 'dorian', age: 37}]

Creating Arrays

Arrays can be created using array literals, the spread operator on an iterable object, the Array() constructor, and the Array.of() and Array.from() methods. 


                
// array literal

                
let count = [1, 2, 3, 4]

                
let names = ['regina', 'dorian', 'jon', 'natalie']

The spread operator does what its name implies and spreads individual pieces of an iterable object out into individual items. In the example below define an array, num and then another, moreNum which spreads the contents of num out as the first 3 elements and then adds 4, 5, and 6. 


                
let num = [ 1, 2, 3]

                
let moreNum = [ ...num, 4, 5, 6] // [ 1, 2, 3, 4, 5, 6] 

                

                
// num is the first entry in this array, but not the individual elements of num

                
 let wrongNum = [ num, 4, 5, 6] // [[ 1, 2, 3], 4, 5, 6]

Passing no arguments to the Array() constructor creates an empty array. Passing a single number as an argument will create and return an array of that length. Passing multiple arguments creates and returns an array with the arguments as its elements. 


                
let none = new Array() // []

                
let empty = new Array(10) // [<10 empty items>] 

                
let scores = new Array(65.6, 78.9, 73.9, 84) // [65.6, 78.9, 73.9, 84]

Passing a single numeric argument into Array() creates an array of that length; passing multiple numeric arguments creates an array that uses these values as elements. This means that one can never create an array with a single numeric element using the Array() constructor.

Array.of() solves this problem by creating and returning an array that uses its arguments as values 


                
let scores = new Array.of(10) // [10]

                
let collection = new Array.of('say words', false, 3.14) // ['say words', false, 3.14]

Array.from() takes as its first argument an iterable or array-like object. With an iterable as an argument, it works like the spread operator. With an array-like object Array.from() creates a true array copy.

JSON

JSON (JavaScript Object Notation) is a format used to store data structures as a string of characters for saving or transmission; this is also known as a data-interchange format. As the name implies, the JSON format resembles javascript objects. The difference is subtle, but notice in JSON formatting the property values are surrounded by double quotes. In JSON, property names must be double-quoted strings. 


              

                // JSON format 

                { 

                   "Code": "EDT", 

                   "Name": "America/New_York", 

                   "GmtOffset": -4, 

                   "IsDaylightSaving": true, 

                   "NextOffsetChange": "2021-11-07T06:00:00Z" 

                }
              

            

              

                // JavaScript Object 

                { 

                   Code: 'EDT', 

                   Name: 'America/New_York', 

                   GmtOffset: -4, 

                   IsDaylightSaving: true, 

                   NextOffsetChange: '2021-11-07T06:00:00Z' 

                }
Why Not use JavaScript Objects?

JSON is a format for serializing data; this means taking some complex in-memory data structure and converting it to a format, such as a string of bytes or characters, that making it easier to transmit or store. Moreover, JSON, while based on JavaScript, is distinct from it and can be used by other languages.

JSON Methods

JSON.parse() and JSON.stringify() are built-in Javascript methods to convert JSON strings into values or objects and values or objects into JSON strings, respectively. They're opposite and complimentary functions for handling JSON data. The typeof operator returns a string of the type of its operand. console.log(typeof 'dorian') // string 


                

                  // JavaScript Object to JSON String

                  const time = { 

                     Code: 'EDT', 

                     Name: 'America/New_York', 

                     GmtOffset: -4, 

                     IsDaylightSaving: true, 

                     NextOffsetChange: '2021-11-07T06:00:00Z', 

                  }
                

                
                
const jsonString = JSON.stringify(time)

                
console.log(jsonString) // {"Code":"EDT","Name":"America/New_York","GmtOffset":-4,"IsDaylightSaving":true,"NextOffsetChange":"2021-11-07T06:00:00Z"}

                
console.log(typeof jsonString) // string

                

                
// JSON string to JavaScript Object

                
const revert = JSON.parse(jsonString) 

                  
console.log(revert) // prints an object like our time variable above

                  
console.log(typeof revert) // object
